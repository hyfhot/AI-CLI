# AI-CLI 版本更新日志

## v2.2.9 (2026-02-28)

### ✨ 新功能
- **自动 PATH 管理**：工具安装后自动添加到环境变量
  - 自动搜索工具可执行文件位置（支持 9 个常见安装目录）
  - 智能检测目录是否已在 PATH 中，避免重复添加
  - 自动更新用户环境变量并刷新当前会话
  - 无需手动配置或重启终端即可使用新工具
  - PATH 长度检查（2047 字符限制）防止系统问题

### 🔧 技术改进
- 新增 `Find-ToolExecutable` 函数：搜索工具可执行文件
- 新增 `Add-ToUserPath` 函数：安全地添加目录到 PATH
- 新增 `Update-PathAfterInstall` 函数：安装后自动更新 PATH
- 支持 .exe、.cmd、.bat 多种可执行文件格式
- WSL 环境自动跳过（由包管理器处理）

## v2.2.0 (2026-02-26)

### ✨ 新功能
- **树状结构管理**：项目支持文件夹分类和多层级组织
  - 支持创建文件夹和项目两种类型
  - 文件夹可包含项目和子文件夹
  - 面包屑导航显示当前位置
  - 递归删除文件夹及其所有子项
- **删除功能**：支持删除项目和文件夹
  - 需要手动输入名称确认删除
  - 删除文件夹时显示包含的子项数量
  - 防止误删除的安全机制
- **配置文件分离**：配置文件从程序目录迁移到用户配置目录
  - 用户配置：`%APPDATA%\AI-CLI\config.json`
  - 默认配置：程序目录的 `config.json`
  - 读取优先级：优先用户配置，不存在则读默认配置
  - 所有修改保存到用户配置
  - 卸载程序时配置不会丢失
- **参数兼容性增强**：支持 `-` 和 `--` 两种参数前缀
  - 例如：`-Init` 和 `--init` 都可以使用
  - 参数不区分大小写
- **帮助功能**：新增 `--help` 参数，显示所有支持的命令和说明

### 🔄 改进
- 安装时自动复制默认配置到用户目录
- 卸载时完整清除程序文件夹
- 自动迁移旧版本平面配置到树状结构
- 优化 UI 显示，使用图标区分文件夹和项目

### 📚 文档更新
- 更新配置文件位置和优先级说明
- 添加配置持久化说明
- 添加参数前缀兼容性说明
- 添加帮助命令使用说明
- 添加树状结构使用说明
- 更新快捷键说明

### 安装方式

#### 快速安装
```powershell
irm https://raw.githubusercontent.com/hyfhot/AI-CLI/master/install.ps1 | iex
```

#### 手动安装
```powershell
# 下载项目
git clone https://github.com/hyfhot/AI-CLI.git
cd AI-CLI

# 初始化配置
.\ai-cli.ps1 -Init

# 运行
.\ai-cli.ps1
```

### 配置文件

#### config.json 结构
```json
{
  "projects": [
    {
      "name": "项目名称",
      "path": "项目路径",
      "description": "项目描述"
    }
  ],
  "tools": [
    {
      "name": "工具命令",
      "displayName": "显示名称",
      "winInstall": "Windows 安装命令",
      "wslInstall": "WSL 安装命令",
      "checkCommand": "检测命令",
      "url": "官方网站"
    }
  ],
  "settings": {
    "language": "auto",
    "defaultEnv": "wsl",
    "terminalEmulator": "default"
  }
}
```

## v1.x (历史版本)

### 特性
- ✅ WinForms 图形界面
- ✅ 环境变量配置（AI_PROJECTS）
- ✅ 基础工具检测和启动
- ✅ Windows/WSL 双环境支持
- ✅ 动态页签命名

### 限制
- ❌ 启动速度慢（GUI 加载）
- ❌ 每次启动后程序退出
- ❌ 不支持多页签
- ❌ 无工具安装功能
- ❌ 配置分散（环境变量 + JSON）

### 迁移到 v2.0

v1.x 用户无需手动迁移，v2.0 会自动：
1. 读取 `AI_PROJECTS` 环境变量
2. 生成 `config.json`
3. 保留所有项目配置
