# Git 提交规范技能

## 触发条件

当用户要求：
- 生成提交说明
- 创建 git commit
- 提交代码变更
- 写 commit message

## 版本号规则

1. **获取最新版本号**
   ```bash
   git log --oneline -1
   ```

2. **版本号格式**: `v{主版本}.{次版本}.{修订版本}`

3. **递增规则**: 修订版本号 +1
   - 示例：`v2.2.3` → `v2.2.4`

## 提交说明格式

```
{版本号}: {修改内容描述}
```

## 执行步骤

1. **读取提交历史**
   ```bash
   git log --oneline -5
   ```

2. **解析版本号**
   - 提取最新版本号
   - 修订版本号 +1

3. **分析变更内容**
   - 使用 `git status` 查看变更文件
   - 使用 `git diff --cached` 查看具体修改
   - 生成简洁的中文描述

4. **生成提交命令**
   ```bash
   git commit -m "vX.Y.Z: 描述"
   ```

## 描述前缀参考

| 类型 | 前缀 | 示例 |
|------|------|------|
| Bug 修复 | 修复... | `v2.2.4: 修复空指针异常` |
| 新功能 | 实现.../增加... | `v2.2.4: 实现用户认证` |
| 优化 | 优化.../改进... | `v2.2.4: 优化启动速度` |
| 文档 | 更新.../添加... | `v2.2.4: 更新 API 文档` |
| 配置 | 更新配置/调整... | `v2.2.4: 更新构建配置` |

## 注意事项

- 描述使用**中文**（与项目现有风格一致）
- 描述应**简洁明了**，概括核心修改
- 如果有多个不相关的修改，建议分多次提交
