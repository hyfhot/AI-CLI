# AI-CLI Project - Claude Code 配置

## Git 提交规范

### 版本号规则

- 版本号格式：`v{主版本}.{次版本}.{修订版本}`
- 新版本号必须在最后一个提交记录的基础上 **+1**
- 从 git 历史中获取最新版本号：`git log --oneline -1`

### 提交说明格式

```
{版本号}: {实际修改内容的描述}
```

**示例：**
```
v2.2.3: 修复用户认证模块的空指针异常
v2.2.2: 实现文档多语言支持并优化目录结构
v2.2.1: 修正工具选择界面环境标签显示位置
```

### 生成提交说明的步骤

1. **读取最近的提交历史**，确定当前最新版本号
   ```bash
   git log --oneline -5
   ```

2. **解析最新版本号**，递增修订版本号
   - 当前最新：`v2.2.2` → 新版本：`v2.2.3`

3. **分析本次变更内容**，生成简洁的描述
   - 描述应使用中文（与项目现有风格一致）
   - 描述应简洁明了，概括核心修改

4. **生成提交命令**
   ```bash
   git commit -m "v2.2.3: {修改描述}"
   ```

### 变更类型参考

| 类型 | 描述前缀 | 示例 |
|------|----------|------|
| Bug 修复 | 修复... | `v2.2.3: 修复空文件夹无法新增项目的问题` |
| 新功能 | 实现.../增加... | `v2.2.0: 实现树状结构管理和删除功能` |
| 改进/优化 | 优化.../改进... | `v2.2.1: 优化新增项目界面` |
| 文档更新 | 更新.../添加...文档 | `v2.2.2: 实现文档多语言支持并优化目录结构` |
| 配置修改 | 更新配置/调整... | `v2.2.0: 实现配置文件优先级读取` |

## 项目信息

- **项目名称**: AI-CLI Launcher
- **项目类型**: PowerShell 终端启动工具

## 常用命令

### 查看提交历史
```bash
git log --oneline -10
```

### 获取最新版本号
```bash
git log --oneline -1 | grep -oE 'v[0-9]+\.[0-9]+\.[0-9]+'
```

### 提交变更
```bash
git add .
git commit -m "vX.Y.Z: 提交说明"
```
